### 3. ABCs of AI playground (internal test tool)

- **Backend:** New route `POST /api/abc` — requires auth and `OPENAI_API_KEY`. Accepts JSON: `prompt` (required), optional `system` (persona/role), `model`, `temperature`, `max_tokens`, `top_p`, `frequency_penalty`, `presence_penalty`, `stream`. Builds messages with optional system message then user message; calls OpenAI with provided params. Returns `{ "content": "..." }`. Does not use `OPENAI_EDIT_MODEL` from env; model comes from request body.
- **Frontend:** New page at `/studio/abc` titled “The ABCs of AI.” Prompt textarea; settings with title, short explanation, dropdown, and “Value used” box (shows default until user selects from dropdown): Model, Temperature, Max tokens, Top P, Frequency penalty, Presence penalty, Stream. **Persona (role):** dropdown of 10 personas plus “None”; each persona sets a system message (Editor, Linguistics expert, Expert software developer, Historian, Science communicator, Teacher, Critical reviewer, Storyteller, Diplomat, Data analyst). Send prompt → response shown with inline markdown.
- **Error handling:** On network error (e.g. “Failed to fetch”), show a clear message: ensure backend is running on localhost or correct `NEXT_PUBLIC_API_URL` / use proxied backend.
- **Nav:** ABCs of AI was added to studio nav then **removed** so the tool is internal only; page remains at `/studio/abc` for direct access.

### 4. Fix: persona (system) in request body

- **Issue:** ABC request body was missing `system` (persona), so the selected role was not sent.
- **Fix:** Include `system: PERSONAS.find(...).systemPrompt` in the JSON body so the backend receives the persona as the system message.
